## 1. 文档目的

这份文档只定义 `P0`。

这里的 `P0` 不是“所有未来可能会用的数据”，而是 `V1 必须先采到、先落库、先校验` 的最小数据集合。没有这些数据，V1 不应该进入正式预测和晨报发布。

P0 只服务以下范围：

1. 研究区域：`山东`
2. 重点传导区域：`京津冀`、`河南`、`江苏`
3. 研究品种：`92#汽油`、`0#柴油`
4. 预测周期：`T+0`、`T+1`、`T+3`
5. 输出能力：`趋势`、`点位`、`区间`、`裂解价差/区域基差`、`预警`、`晨报`

## 2. P0纳入原则

只有满足下面至少一条，数据才进入 P0：

1. 直接决定 `山东地炼出厂价` 或 `山东主营批发价` 的日度判断
2. 直接决定 `区域到站/落地价` 的传导与套利判断
3. 能触发 `黄/橙/红` 级预警
4. 是晨报、盘中快评、对话式再预测的必需输入

P0 默认按以下优先级采集：

1. `官方接口 / 官方公告`
2. `商业授权数据`
3. `内部系统 / 人工录入`
4. `公开网页爬虫`

## 4. 数据字典

## 4.1 山东地炼出厂报价 `sd_refinery_quote`

### 业务定义

V1 的第一主预测对象。要求同口径、可追溯、尽量贴近真实可成交出厂价。

### 关键字段

| 字段名 | 类型 | 必填 | 示例 | 说明 | 落表 |
|---|---|---:|---|---|---|
| `trade_date` | `date` | Y | `2026-05-29` | 交易日 | `fact_market_timeseries.dt` |
| `observation_time` | `timestamptz` | Y | `2026-05-29 11:30:00+08` | 报价观察时间 | `fact_market_timeseries.observation_time` |
| `publish_time` | `timestamptz` | Y | `2026-05-29 11:34:00+08` | 来源发布时间 | `fact_market_timeseries.publish_time` |
| `refinery_code` | `varchar(64)` | Y | `JINGBO` | 炼厂编码 | `dim_entity.entity_code` |
| `refinery_name` | `varchar(128)` | Y | `京博石化` | 炼厂名称 | `dim_entity.entity_name` |
| `product_code` | `varchar(32)` | Y | `GAS92` | 品种编码 | `dim_entity.product_family` / 业务映射 |
| `quote_price_yuan_ton` | `numeric(12,2)` | Y | `6880.00` | 出厂报价 | `fact_market_timeseries.value_num` |
| `low_price_yuan_ton` | `numeric(12,2)` | N | `6860.00` | 区间下沿 | 原始 `payload` |
| `high_price_yuan_ton` | `numeric(12,2)` | N | `6900.00` | 区间上沿 | 原始 `payload` |
| `tax_flag` | `varchar(16)` | Y | `with_tax` | 含税/不含税 | 原始 `payload` |
| `pickup_mode` | `varchar(32)` | N | `ex_works` | 提货方式 | 原始 `payload` |
| `payment_term` | `varchar(64)` | N | `cash` | 账期/付款方式 | 原始 `payload` |
| `sample_quality` | `varchar(16)` | Y | `ok` | 质量标记 | `fact_market_timeseries.quality_flag` |
| `source_record_id` | `varchar(128)` | Y | `jingbo_20260529_1130_gas92` | 源记录唯一键 | `fact_market_timeseries.source_record_id` |

### 来源优先级

1. 商业授权数据
2. 业务群采样+人工录入
3. 内部报价单

### 采集频率

- `08:30`
- `11:30`
- `14:30`
- `17:00`
- 盘中重大异动时允许加采

### 校验规则

1. 同一炼厂同一品种相邻两次报价绝对变动 `> 300 元/吨` 时标记 `outlier`
2. 同一时点 `low_price <= quote_price <= high_price`
3. 同一来源同一 `source_record_id` 幂等写入
4. 未明确含税口径的样本不得进入主预测特征

## 4.3 区域到站/落地价 `regional_landed_price`

### 业务定义

不是展示价格，而是 `山东出厂价 + 运费 + 票据/账期成本 + 区域基差` 的落地价格真值或近真值。

### 覆盖范围

V1 只采：

1. `京津冀`
2. `河南`
3. `江苏`

### 来源优先级

1. 内部成交
2. 业务录入
3. 授权估价

1. `landed_price >= sd_refinery_quote + freight - 100`

## 6. 数据质检规则

## 6.1 通用规则

1. 每条记录必须同时具备 `source_id`、`source_record_id`、`ingest_time`
2. 所有时间统一入库为 `timestamptz`
3. 原始值永不覆盖，只允许新版本写入
4. 不能用文本默认值替代数值缺失

## 6.2 价格规则

1. 同口径才可比较
2. 同口径才可拼接时间序列
3. 出厂价、批发价、到站价、零售价严禁混用

## 6.3 新闻规则

1. 原油新闻与成品油新闻必须分 topic
2. 事件未确认时只能进入观察池
3. 事件聚类必须保留 `cluster_id`

## 6.4 预警规则

1. `orange` / `red` 事件自动推送人工复核
2. 价格跳变超阈值自动生成数据冲突工单

## 7. 现阶段阻塞项

1. 山东地炼主链仍依赖授权数据或人工采样，不能只靠爬虫
2. 主营批发价需要合作接口或企业账户
3. 区域落地价需要内部成交与业务录入配合
4. 开工、库存、主营投放目前仍有较重人工维护成分

## 8. 开工建议

按顺序做，不要并行发散：

1. 先把 `Brent / 调价公告 / 省级限价 / 成品油新闻` 采集任务稳定
2. 再把 `山东地炼报价 / 主营批发价 / 区域落地价` 的人工导入模板定死
3. 最后把 `开工库存 / 物流运费 / 内部成交` 接进预测主链

只要这一步做完，预测系统就有可跑的 P0 数据底座。
