# 设计稿：国内成品油研究智能体系统 V1

生成日期：2026-05-28

## 1. 目标定位

这不是一个“原油观点机”，而是一个把 `Brent研究 -> 国内成品油定价 -> 采销存动作` 串起来的研究决策支持系统。

V1 的目标不是一次性做成全国全品种全链条平台，而是先做出一个稳定、可解释、可回测、可落地的最小闭环：

1. 看懂今天 Brent、国内成品油市场、政策机制和异常事件在驱动什么
2. 把这些驱动转换成对国内成品油有意义的桥接因子
3. 对 `趋势、点位、区间、裂解价差` 形成确定性预测输出
4. 把预测翻译成研究结论、预警和采销动作建议

一句话定义：

`主控智能体 + 专家子智能体 + 确定性打分/校准引擎 + 审计留痕前端`

## 2. 10 位专家 panel 共识

本方案已按 10 个专家视角收敛，结论高度一致：

1. 原油研究专家：Brent 不能直接映射成 92#/0#，必须先转成 `成本推动 + 调价传导 + 现货强弱`
2. 国内成品油现货专家：V1 不要做“全国统一预测器”，先围绕 `山东定价信号 + 区域传导 + 区域基差`
3. 量化建模专家：主路线必须是 `业务打分模型 + 统计校准`，LLM 不能直接决定数值预测
4. 数据平台专家：先做 `point-in-time correct` 数据底座，保证同因子同结果
5. Agent 架构专家：采用 `星型编排`，主控裁决，禁止 agent 自由群聊
6. 前端产品专家：首页应该是 `价格 + 区域 + 驱动 + 观点 + 异常` 的决策入口
7. 风控审计专家：系统必须支持 `一键回放`，人工修正不能覆盖审计
8. 评估治理专家：先分清 `趋势/点位/区间/裂解价差` 四类对象，再建基线和上线闸门
9. 新闻事件专家：新闻要收敛成 `事件本体 + 状态机 + 半衰期`，不要做泛情绪分
10. 业务落地专家：V1 先服务 `采、销、存、价` 四个动作，不做全能研究平台

## 3. 现有输入与边界

当前已确认可用输入：

1. Wind Brent 实时价格
   - 语雀页面元数据可确认文档名为《Wind API 接口调用说明》
   - 可提取到的描述信息显示基础服务地址为 `<INTERNAL_WIND_API_URL>`
   - 元数据中可见返回格式为 `JSON`，并提到 `/price` 接口
   - 这里的接口字段细节仍建议后续按完整文档再补齐

2. Brent 预测日报 API
   - 地址：`<BRENT_REPORT_API_URL>`
   - 2026-05-28 返回的 `reportDate` 为 `2026-05-28`
   - 内容包含：Brent 当前价格、方向、点位、区间、支撑阻力、月差、裂解、事件扫描、风险提示

3. Jinshi 新闻 skill
   - 本地已存在 `jinshi-news-query`
   - 返回样例字段已确认包含：
     - `publish_time`
     - `title`
     - `content`
     - `direction_hint`
     - `direction_strength`
     - `relevance_score`
     - `major_event_candidate`
     - `major_score`

4. 业务规则文档
   - `山东成品油市场价预测打分模型.docx`
   - 已明确覆盖：
     - 预测对象：山东 `92#汽油`、`0#柴油`
     - 周期：`T+1 / T+3 / T+7 / 次月`
     - 核心维度：成本、供给、需求、库存、情绪、政策预期

结论：

V1 已经具备“先搭起来”的基础，但这些输入还不等于 V1 的完整主输入。

必须明确：

1. `Wind + Brent 日报 + Jinshi 原油新闻` 只构成 `上游成本与黑天鹅输入层`
2. 真正驱动国内成品油日度价格判断的主输入，还必须补齐：
   - 山东 / 全国 / 区域市场价格
   - 主营批发价 / 地炼出厂价
   - 地炼与主营开工率
   - 产销率
   - 库存
   - 检修 / 复产计划
   - 物流与运费
   - 调价窗口与政策公告
   - 成品油市场新闻流
3. 因此，Jinshi 不能作为 V1 唯一新闻源，只能作为 `上游原油事件辅助源`

## 4. 核心路线选择

### 4.1 不选什么

不选“把全部因子直接喂给 LLM 输出价格预测”作为生产主引擎。

原因：

1. 你有硬约束：`相同输入必须得到相同输出`
2. 需要可回测、可冻结、可审计
3. 需要对业务解释“为什么是这个结论”
4. 需要支持人工修正、风控、复盘

纯 LLM 直推不满足这些要求。

### 4.2 选什么

V1 主路线：

`业务打分模型 -> 统计校准 -> 结构化输出`

LLM 只做三件事：

1. 非结构化信息抽取
2. 因子归因和解释文本生成
3. 报告草稿和问答辅助

LLM 不直接做：

1. 最终方向判断
2. 分值和权重赋值
3. 点位和区间计算
4. 风险限额或业务动作裁决

## 5. V1 业务切口

V1 先聚焦一个小而硬的切口，不做全国统一预测器。

### 5.1 预测范围

优先预测：

1. 山东 `92#汽油`
2. 山东 `0#柴油`
3. 山东 `95#汽油` 作为扩展项

优先展示：

1. Brent 实时价格
2. `92# 全国汽油`
3. `95# 全国汽油`
4. `0# 全国柴油`
5. 区域热力图
6. 山东与华东、华北、华中的区域基差

### 5.2 业务目标

V1 最先服务四个动作：

1. 什么时候采
2. 什么时候卖
3. 库存怎么摆
4. 异常行情如何统一口径

## 6. 系统总体架构

建议采用七层架构：

1. 数据接入层
2. 标准化与特征层
3. 事件因子层
4. 预测引擎层
5. 预警引擎层
6. Agent 编排层
7. 应用与审计层

### 6.1 数据接入层

V1 的数据输入必须分层，不能继续把“原油输入”和“成品油主输入”混在一起。

#### 6.1.1 上游原油层

1. Wind Brent 实时价格
2. Brent 日报 API
3. Jinshi 原油新闻

作用：

1. 上游成本锚
2. 地缘与黑天鹅预警
3. 原油方向与波动率输入

#### 6.1.2 国内成品油结构化数据层

这是 V1 预测主输入，优先级高于原油新闻。

1. 山东 / 全国 / 区域市场价
2. 主营批发价 / 地炼出厂价
3. 地炼与主营开工率
4. 产销率
5. 库存
6. 检修 / 复产计划
7. 物流与运费
8. 汇率

#### 6.1.3 国内成品油市场新闻层

这是成品油研究的日度驱动新闻主链。

1. 地炼 / 主营调价
2. 检修、复产、降负荷
3. 出货节奏
4. 区域资源流向
5. 物流受阻
6. 市场情绪与促销

#### 6.1.4 政策与制度层

1. 调价窗口日历
2. 发改委调价公告
3. 税费、出口配额、安监环保、运输政策

#### 6.1.5 内部经营层

长期建议接入：

1. 内部采购成交
2. 内部销售报价
3. 库存与周转
4. 客户询盘与订单节奏

### 6.2 标准化与特征层

采用 `ODS -> DWD -> DWS -> Feature Snapshot -> Serving`

关键原则：

1. 每条数据必须区分：
   - `observation_time`
   - `publish_time`
   - `ingest_time`
2. 所有预测和回测一律按 `as_of_time` 消费数据
3. 保留修订历史，不做覆盖写
4. 特征物化，支持按版本冻结

最小核心表：

1. `dim_indicator`
2. `dim_entity`
3. `dim_source`
4. `fact_market_timeseries`
5. `fact_forecast_snapshot`
6. `fact_news_event`
7. `feature_registry`
8. `feature_snapshot`
9. `feature_materialization_log`

## 7. 事件因子层

Jinshi 和 Brent 日报中的事件，只能作为上游事件层；成品油市场新闻和政策公告也必须进入同一个事件状态机。

### 7.1 事件卡字段

每个事件至少产出：

1. `event_id`
2. `event_type`
3. `event_time`
4. `entities`
5. `geo_scope`
6. `commodity_scope`
7. `direction`
8. `intensity`
9. `confidence`
10. `time_to_effect`
11. `half_life`
12. `state`

### 7.2 V1 只做 6 类事件

1. 原油供给扰动
2. 炼厂开工/检修/复产
3. 国内成品油调价与政策机制
4. 成品油贸易、出口配额与监管政策
5. 航运与物流扰动
6. 终端需求冲击

### 7.3 事件处理机制

1. 去重：按 `实体 + 动作 + 地点 + 时间窗` 聚类
2. 状态：`rumored / confirmed / revised / canceled`
3. 衰减：按事件类别设半衰期
4. 冲突：允许共存，由主控或风控 agent 做裁决

补充原则：

1. 原油新闻只负责解释上游冲击
2. 成品油市场新闻优先用于日度市场状态判断
3. 政策类事件优先级高于市场传闻
4. 黑天鹅事件触发后，不直接覆盖预测，而是交给独立预警引擎处理

## 8. 预测引擎设计

### 8.1 预测对象

四类对象分开建模：

1. 趋势
2. 点位
3. 区间
4. 裂解价差

### 8.2 核心桥接变量

Brent 研究结果先转成三类桥接变量，再进入成品油预测：

1. `成本推动分数`
2. `调价传导分数`
3. `现货强弱分数`

再按品种拆：

1. `汽油修正项`
2. `柴油修正项`

### 8.3 主引擎

主引擎推荐为：

`单因子标准化 -> 子维度得分 -> 总分聚合 -> 概率/点位/区间映射`

其中：

1. 规则骨架沿用业务文档
2. 权重和阈值用历史样本校准
3. 点位用统计校准层映射
4. 区间由波动率、残差分布、历史误差带给出
5. 裂解价差单独建模，不和单边价格混在一起

### 8.4 推荐建模方式

V1 推荐：

1. 规则化打分引擎
2. 有序逻辑回归 / Logistic
3. 单调约束 GBDT 或 LightGBM 作为校准层

不推荐 V1 直接上黑箱端到端模型。

## 9. 预警引擎设计

预警引擎独立于预测引擎运行，负责识别“非常态事件”，而不是负责日常价格判断。

### 9.1 预警引擎职责

1. 识别黑天鹅、地缘、突发政策、炼厂事故、物流中断
2. 判断这些事件是否足以打断日度预测逻辑
3. 触发短周期重算、降级、人工复核和业务提醒

### 9.2 预警触发器

分三类：

1. 价格触发
   - Brent 短时波动超阈值
   - 山东 / 区域现货价格跳变超阈值
   - 区域基差、裂解价差急变
2. 事件触发
   - 地缘冲突
   - 非计划停车
   - 港口 / 物流中断
   - 临时调控或重大政策变化
3. 一致性触发
   - 模型结论与市场实际异动明显冲突
   - 上游原油与国内现货传导失真
   - 旧观点已被新事件推翻

### 9.3 预警级别

1. 蓝色：关注级
2. 黄色：影响级
3. 橙色：重算级
4. 红色：黑天鹅级

### 9.4 预警动作

1. 仅重算 `D1 / D3`
2. 从单点预测切到 `区间 + 情景`
3. 下调旧观点置信度
4. 强制人工复核
5. 向采购、销售、管理层推送不同版本提醒

## 10. 多智能体架构

V1 采用 `主控 + 专家 agent + 风控/证据 agent`。

### 9.1 核心 agent team

建议 6 个核心 agent：

1. `MacroPolicyAgent`
   - 原油大势、地缘、宏观、国内政策

2. `SupplyAgent`
   - 开工率、检修、供应、库存补库/去库

3. `DemandAgent`
   - 出行、物流、基建、工矿、农业、季节性

4. `MarketStructureAgent`
   - 裂解价差、批零价差、区域基差、盘面/现货结构

5. `QuantForecastAgent`
   - 分值聚合、概率、点位、区间计算

6. `EvidenceRiskAgent`
   - 证据校验、时效检查、冲突提示、降级建议

可选增强：

7. `RegionalAgent`
8. `RefereeAgent`

### 9.2 编排规则

1. 主控先拆任务，不自己重做研究
2. 所有子 agent 并行、独立产出
3. 通信采用星型，不允许自由群聊
4. 每个 agent 最多两轮输出
5. 所有结论必须绑定证据、时间、适用范围、置信度

### 9.3 统一输出格式

每个 agent 统一返回：

1. `claim`
2. `drivers`
3. `evidence`
4. `confidence`
5. `horizon`
6. `risk_flags`
7. `conflict_with`

3. 趋势、点位、区间、裂解价差
4. 因子归因
5. 失效条件

### 11.2 业务输出

1. 采购建议
2. 销售建议
3. 库存风险等级
4. 预警触发器

## 13. 风控与审计

### 12.1 必须保留的留痕

1. 输入留痕
2. 模型与规则版本留痕
3. 中间推理留痕
4. 输出留痕
5. 人工修正留痕
6. 权限与分发留痕

### 12.2 人工修正机制

分三级：

1. `L1` 备注修正
2. `L2` 区间修正
3. `L3` 结论覆盖

其中 `L3` 必须双人审批。

### 12.3 降级机制

以下情况自动降级：

1. 数据异常
2. 事件冲突未解
3. 模型漂移超阈值
4. 极端波动

降级后：

1. 只输出方向和区间
2. 降低置信度
3. 强制人工复核
4. 暂停报价参考输出

## 14. 评估体系

### 13.1 四类对象的核心指标

1. 趋势
   - Accuracy
   - F1 / Macro-F1
   - Balanced Accuracy

2. 点位
   - MAE
   - RMSE
   - Bias
   - 容忍带命中率

3. 区间
   - Coverage
   - Width
   - Interval Score

4. 裂解价差
   - Spread MAE
   - 方向命中率

### 13.2 三层基线

1. 朴素基线
2. 统计基线
3. 业务基线

### 13.3 上线闸门

必须同时满足：

1. 优于朴素基线
2. 不弱于统计基线
3. 关键业务场景优于人工/规则基线

## 16. V1 最终建议

第一版最稳的落地定义是：

`以山东 92#/0# 为预测核心，以 Brent 上游层 + 成品油结构化数据层 + 成品油市场新闻层 + 政策制度层 + 独立预警引擎为主链路，以区域热力图和采销动作建议为业务出口。`

不要把 V1 做成“万能平台”。

先把这 5 件事做到位：
