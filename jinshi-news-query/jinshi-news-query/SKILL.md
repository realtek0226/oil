# JinShi News Query Skill

Query JinShi financial news data through the Chambroad AGI API with automatic AES key encryption.

## Quick Start

```bash
python scripts/query.py
```

Default: fetches last 2 days of news.

## Usage

### Fetch last 2 days (default)
```bash
python scripts/query.py
```

### Fetch specific date range
```bash
python scripts/query.py --start "2026-03-17" --end "2026-03-18"
```

### With time precision
```bash
python scripts/query.py --start "2026-03-17 17:00:00" --end "2026-03-18 23:00:00"
```

### Output to file
```bash
python scripts/query.py --start "2026-03-17" --end "2026-03-18" --output news.json
```

## Key Encryption

The `key` field is an AES-128-CBC encrypted timestamp with a zero IV, prepended with the IV, then Base64 encoded:

1. **Secret Key** (Base64): `K3nlUWKurKarvRWICc9PPA==` (decodes to 16 bytes)
2. **Plaintext**: Current time formatted as `%Y-%m-%d %H:%M:%S`
3. **Algorithm**: AES-128-CBC with PKCS7 padding, IV = `b'\x00' * 16`
4. **Output Format**: `Base64(IV + Ciphertext)`
5. **Validity**: 10 minutes from encryption time

The script handles encryption automatically — no manual key generation needed.

## Request Format

```json
{
    "inputs": {
        "type": "数据查询",
        "key": "<aes_encrypted_timestamp>",
        "startdate": "2026-03-17 17:00:00",
        "enddate": "2026-03-18 23:00:00"
    },
    "response_mode": "blocking",
    "user": "jinshi-news-query-skill"
}
```

- `startdate`/`enddate`: Optional. Omit for default 2-day window.
- `key`: Auto-generated by script.

## Troubleshooting

| Issue | Solution |
|---|---|
| `ModuleNotFoundError: Crypto` | `pip install pycryptodome` |
| `密钥校验异常` | Verify encryption: MUST use AES-CBC with Zero IV, and send `Base64(IV + Ciphertext)` |
| Server error `time data ':XX' does not match format` | CBC Block 2 decrypted correctly (IV-independent), but Block 1 failed. Indicates wrong IV or missing IV prepend |
| Key expired error | System clock may be off; check `date` command |
| Empty response | Verify date range has data; try default (no dates) |

## Key Pitfalls (Debugging Notes)

The following were discovered through extensive trial-and-error — avoid repeating these mistakes:

1. **ECB mode fails**: `AES-128-ECB` returns `密钥校验异常`. The API strictly requires `AES-128-CBC`.
2. **IV must be prepended**: Sending just `Base64(ciphertext)` fails with key validation errors. The API expects `Base64(IV + ciphertext)` where IV is 16 zero bytes.
3. **Server error decoding**: If the server returns `ValueError: time data ':XX' does not match format`, it means CBC Block 2 decrypted correctly (which is IV-independent in CBC), but Block 1 failed. This confirms the key and mode are correct, but the IV is wrong or missing.
4. **Base64 vs Hex**: The API expects Base64 encoding of the raw bytes, not hex strings.
