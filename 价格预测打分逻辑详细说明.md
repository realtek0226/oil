# 价格预测打分逻辑详细说明

本文说明当前系统如何对山东 92# 汽油价格进行研判。当前系统不是让大模型直接猜价格，也不是把分数乘一个人工设定的点值，而是按以下链路生成结果：

```text
数据取数 → 规则智能体打分 → 形成综合分/业务分 → 分数进入状态桶 → 查历史同状态真实涨跌分布 → 生成点位、区间、可靠度和经营建议
```

## 一、当前系统输出什么

| 输出 | 含义 |
| --- | --- |
| 预测点位 | 对山东 92# 汽油未来目标周期价格的参考中枢 |
| 预测区间 | 围绕参考中枢给出的经营参考范围 |
| 方向判断 | 上涨、下跌、震荡等展示口径 |
| 研判可靠度 | 本次研判作为经营参考的质量评分，不是价格命中概率 |
| 经营建议 | 根据预测变化、风险等级、调价窗口和主驱动生成的经营动作建议 |

当前支持四个周期：

| 周期 | 含义 |
| --- | --- |
| D1 | 下一交易日 |
| D3 | 未来 3 个交易日 |
| W1 | 未来 1 周 |
| M1 | 未来 1 月 |

## 二、数据取数口径

系统按“早上形成经营参考”的场景控制数据边界：

```text
预测日 T 的研判，只允许使用 T 日 07:00 前已经可见或已经落库的数据。
```

主要数据包括：

| 数据 | 用途 |
| --- | --- |
| 山东 92# 汽油市场价 | 当前价格锚点，也是历史状态映射的目标价格 |
| 全国及各区域 92# 汽油市场价 | 区域价差、净回款价差、跨区外发判断 |
| Brent 结算价、Brent 预测日报 | 原油成本变化、调价预期、裂解价差预测 |
| 汽油裂解价差、MTBE、石脑油、人民币汇率中间价 | 成本和调油支撑判断 |
| 山东地炼开工率、炼油利润、库存 | 供给压力和供应弹性判断 |
| 山东地炼产销率 | 需求承接判断，注意这里看的是产销率，不是单纯销量 |
| 发改委调价窗口和调价预期金额 | 政策周期扰动 |
| 成品油资讯、山东日评、事件快讯 | 炼厂推价、成交情绪、突发风险、黑天鹅预警 |
| 区域运费 | 净回款价差判断 |

如果某项数据缺失，系统不把它解释为“中性”，而是：

```text
缺失字段 → 该项按 0 分处理 → 页面标记缺失 → 降低数据质量和可靠度
```

这样做是为了避免把“没有数据”包装成“市场平衡”。

Brent 预测日报单独做质量门控：

| 场景 | 系统处理 |
| --- | --- |
| 有预测日同日 Brent 日报 | 正常进入成本智能体和 Brent 预测口径 |
| 预测日 Brent 日报缺失 | 标记为 `brent_daily_report_missing`，可靠度压低，D1 区间放宽 |
| 接口返回旧日期 Brent 日报 | 标记为 `brent_daily_report_stale`，不按正常日报口径解释 |

这类降级场景下，点位只能作为观察参考，不应直接触发补库、放量或跨区外发等经营动作。

## 三、两套预测结果

系统保留两套预测结果，二者相互独立。

| 模型 | 作用 | 是否参与对方加权 |
| --- | --- | --- |
| 智能体综合预测 | 系统主预测，综合原油、结构、供需、政策、资讯等因子 | 否 |
| 业务基准打分模型 | 按《山东成品油市场价预测打分模型》转成的规则逐条命中，作为业务口径对照 | 否 |

业务基准打分模型不混入智能体综合分，也不参与综合预测的可靠度。页面会单独展示它的分数、点位和方向，用于和智能体综合预测对比。

## 四、智能体综合预测逻辑

### 1. 规则智能体

当前主预测使用以下规则智能体：

| 智能体 | 主要判断内容 | 当前权重 |
| --- | --- | ---: |
| 原油成本 | Brent 预测涨跌、MTBE、石脑油；裂解价差仅作为观察值 | 22% |
| 供给库存 | 山东地炼开工率、库存、炼油利润 | 20% |
| 需求季节性 | 产销率、月度产销变化、淡旺季、节假日 | 18% |
| 市场结构 | 山东与全国、各区域的价格结构和现货动量 | 16% |
| 政策周期 | 调价窗口、预测调价金额、上轮调价影响 | 12% |
| 成品油资讯 | 成交活跃度、贸易商心态、报价行为 | 12% |
| 山东现货跳变 | 实际成交、炼厂推价、抢货、低价资源扫空等强信号 | 不参与常规加权，只做 D1 点位修正 |
| 事件风险 | 地缘、OPEC、航运、黑天鹅等突发风险 | 不参与常规加权，只做风控和强事件修正 |

每个智能体先输出原始分。需要注意：原始分不是所有智能体都天然加到 -100 到 +100；每个智能体都有自己的“有效满分”。系统合成前先把原始分除以该智能体有效满分，得到 -1 到 +1 的归一化分，再乘专家先验权重。

| 分数方向 | 业务含义 |
| --- | --- |
| 正分 | 对山东 92# 价格偏利多 |
| 负分 | 对山东 92# 价格偏利空 |
| 0 分 | 该项没有明显方向，或数据缺失后按 0 处理 |

系统按专家先验权重合成智能体综合分。综合分保留为 -1 到 +1 的小数；进入状态桶时只做一次单位换算：`分数点 = 综合分 × 100`。这个 ×100 不是再次加权，也不是放大预测力度，只是把 `-0.0676` 写成 `-6.76 分`，便于和状态桶阈值 `-12、-6、-2、2、6、12` 比较。

当前各智能体有效满分如下：

| 智能体 | 原始有效区间 | 归一化方式 | 说明 |
| --- | ---: | --- | --- |
| 原油成本 | -65 到 +65 | 原始分 / 65 | Brent 最高 ±55，MTBE/石脑油最高 ±10；汽油裂解价差分位仅展示，不计分 |
| 市场结构 | -100 到 +100 | 原始分 / 100 | 区域价差 ±65，全国价差 ±20，山东价格动量 ±15 |
| 供给库存 | -62 到 +62 | 原始分 / 62 | 开工率分位 ±40，开工变化 ±12，炼油利润 ±10；库存合计分位当前预留不计分 |
| 需求季节 | 约 -92 到 +98 | 原始分 / 100 | 产销率、月度产销变化、季节性、节假日共同作用，接近 100 分制 |
| 成品油资讯 | -100 到 +100 | 原始分 / 100 | LLM 只抽取标签，成交活跃度、贸易商心态、报价行为由代码固定映射 |
| 政策周期 | 约 -99.5 到 +99.5 | 原始分 / 100 | 调价预测金额、窗口距离、上轮调价共同作用，接近 100 分制 |
| 山东现货跳变 | -100 到 +100 | 原始分 / 100 | 强信号触发后可达到安全截断；不参与常规加权，只修正 D1 点位 |
| 事件风险 | -100 到 +100 | 原始分 / 100 | 高等级事件可达到满格；主要用于风控、区间和人工复核 |

这里的 -100 到 +100 是系统安全截断，不是要求每个智能体的分项都必须凑满 100。对于原油成本、供给库存这类当前有效满分不足 100 的智能体，系统已按有效满分归一化，避免专家权重被低估。

### 2. 成品油资讯和事件的处理

成品油资讯和事件快讯先由 LLM 抽取标签，再由代码按固定规则转成分数。大模型只做标签抽取和原因表述，不直接给价格点位。

| 标签类别 | 示例 | 用途 |
| --- | --- | --- |
| 成交活跃度 | 成交偏强、平稳、转弱 | 进入成品油资讯智能体 |
| 贸易商心态 | 抢货、观望、抛货 | 进入成品油资讯智能体 |
| 报价行为 | 推涨、挺价、让利、下调 | 进入成品油资讯智能体 |
| 事件风险 | 地缘、供应中断、政策突发、天气 | 进入事件风险闸门 |

没有完整历史标签的资讯，不反推历史分数；它会影响当前解释、预警、可靠度和经营建议。

### 3. 智能体综合点位生成

当前点位不再使用旧线性换算，也不使用人工点值上限。智能体综合分先换算成“分数点”，再通过状态桶映射为价格涨跌额：

```text
综合分 = Σ(智能体归一化分 × 专家权重) / Σ启用权重
分数点 = 综合分 × 100
状态桶 = 按分数点命中强空、偏空、弱空、震荡、弱多、偏多、强多
```

P50 的来源不是人工设定，也不是外部报价。它来自系统历史样本中的真实涨跌分布：

```text
1. 取预测日前已经完成的历史样本。
2. 对每个历史日期，用当前规则智能体重新打分，得到该历史日的综合分。
3. 将综合分 × 100 得到历史日分数点。
4. 按分数点把历史日归入强空、偏空、弱空、震荡、弱多、偏多、强多状态桶。
5. 计算该历史日到目标日的真实涨跌：
   真实涨跌 = 目标日山东92#市场价 - 历史日山东92#市场价
6. 取当前预测命中的同状态桶历史样本，把这些真实涨跌从低到高排序。
7. P50 = 排在中间位置的真实涨跌，也就是中位数。
8. 如果当前命中强多、偏多、弱多、强空、偏空或弱空，但整体 P50 被正负样本抵消到 0，系统会检查同方向样本是否足够；足够时用同方向样本的保守分位数作为基础涨跌额。
```

举例：如果本次命中“偏空”桶，选中的历史偏空样本真实涨跌排序后为：

```text
-80、-55、-30、-15、0、20、45
```

那么 P50 就是中间的 `-15`，基础预测涨跌额就是 `-15 元/吨`。

同方向保守分位数只在方向桶被抵消时启用。例如当前命中“强多”桶，但同桶历史涨跌整体 P50 因正负样本混杂被压到 0；如果同桶上涨样本数量达到最低要求，系统不会继续给出 0，而是取上涨样本的 P25 作为保守上涨幅度。强空、偏空、弱空则对应取下跌样本的 P75，也就是更靠近 0 的保守下跌幅度。这样做的目的不是人为放大，而是避免“强方向状态桶”因为历史样本正负抵消而退化成所有周期同一个当前价。

如果当前状态桶样本少于 12 条，智能体综合预测会合并相邻状态桶，直到样本达到最低要求；若仍没有有效样本，才使用专家冷启动状态桶。当前系统还需要注意一个边界：历史分数是用“当前规则”对历史日期重新回放得到的，不等同于当时真实发布过的预测记录。因此 P50 是当前规则下的历史同状态回放结果，后续仍需要用冻结规则做前推验证。

智能体综合分状态桶如下。这里的分数是 `综合分 × 100` 后的分数点，不是业务打分模型的原始总分。

| 智能体综合状态桶 | 分数范围 | 业务含义 |
| --- | ---: | --- |
| 强空 | <= -12 | 多个因素明显压制现货 |
| 偏空 | -12 到 -6 | 下行因素占优 |
| 弱空 | -6 到 -2 | 偏弱但力度有限 |
| 震荡 | -2 到 2 | 方向不明显 |
| 弱多 | 2 到 6 | 偏强但力度有限 |
| 偏多 | 6 到 12 | 上行因素占优 |
| 强多 | >= 12 | 多个因素明显支撑现货 |

点位公式：

```text
基础预测涨跌额 = 当前状态桶历史真实涨跌的 P50 中位数
若方向桶 P50 被抵消为 0 且同方向样本充足：
    多头桶基础涨跌额 = 同桶上涨样本 P25
    空头桶基础涨跌额 = 同桶下跌样本 P75
预测点位 = 当前山东 92# 市场价 + 基础预测涨跌额 + 明确点位修正
```

样本选择：

| 条件 | 系统处理 |
| --- | --- |
| 当前状态桶样本不少于 12 条 | 使用当前桶样本 |
| 当前状态桶样本少于 12 条 | 合并相邻状态桶，直到样本不少于 12 条 |
| 全部相邻桶合并后仍不足 12 条 | 使用可用合并样本，并在页面展示样本量；完全没有可用历史时才使用冷启动专家状态桶 |

方向约束：

| 分数状态 | 约束 |
| --- | --- |
| 负分状态桶 | 基础预测涨跌额不能为正 |
| 正分状态桶 | 基础预测涨跌额不能为负 |
| 震荡状态桶 | 基础预测涨跌额按 0 处理 |
| 方向桶被抵消为 0 | 同方向样本充足时，启用保守同方向分位数；样本不足时仍按 0 处理 |

这个约束解决了过去“业务负分却预测上涨”的问题。若出现山东现货跳变或高等级事件冲击，则作为单独强信号修正，不属于分数映射；但强信号也有触发边界，不能因为缺少成交证据而连续多日重复放大。

### 4. 点位修正

点位修正只处理明确、强烈、可解释的信号，当前主要用于 D1。

| 修正来源 | 触发逻辑 | 影响 |
| --- | --- | --- |
| 山东现货跳变 | 实际成交重心、炼厂报价调整、低价资源扫空、封单惜售、贸易商抢货/抛货、出货节奏等信号明显；或当日急涨但产销率低于 70%；或三日急涨后当日继续回落且产销率低于 70% | 对 D1 点位做上修或下修 |
| 高等级事件冲击 | 事件风险为 high/extreme，Brent 预测涨跌绝对值不低于 3 美元，且事件方向与 Brent 方向一致 | 只做有限对冲修正，不替代成本智能体 |

山东现货跳变的防重复规则：

```text
只看三日累计急涨不够，必须同时看到当日急涨未被产销率验证，或急涨后当日出现实质回落。
```

如果只有产销率偏弱，但没有成交重心、炼厂报价、低价扫空、贸易商抛货、出货承压等硬证据，则作为弱现货提示，不连续多日按跳变信号放大。

事件冲击的防重复规则：

```text
Brent 成本变化已经进入成本智能体；事件智能体主要负责预警、区间放大和经营降级。
只有当事件方向与当前基础点位相反时，才做有限对冲：
high：25 元/吨；extreme：40 元/吨。
若事件方向与基础点位同向，不再额外加点位，避免重复放大。
```

如果没有明确强信号，点位修正为 0。

## 五、业务基准打分模型

业务基准模型来自《山东成品油市场价预测打分模型》转成的结构化规则。它单独输出分数和预测点位，不参与智能体综合加权。

### 1. D1 业务基准规则

| 因子 | 权重/封顶 | 当前判断口径 |
| --- | ---: | --- |
| 成本侧 | 40 | Brent 预测涨跌分档；汽油裂解价差分位当前只作观察字段，不参与 D1 成本侧评分 |
| 供应侧 | 15 | 山东地炼开工率分位，低开工偏利多，高开工偏利空 |
| 需求侧 | 30 | 山东地炼产销率分档，产销率越高越偏利多 |
| 情绪侧 | 15 | 成交活跃度和贸易商心态，成交活跃偏利多，抛货让利偏利空 |

产销率口径：

```text
产销率 = 销量 / 产量 × 100%
```

需求侧当前看产销率，不再用销量环比替代。

### 2. D3 业务基准规则

D3 判断未来 3 个交易日的价格偏向，使用 `horizon_forecasts.D3` 中的 Brent 预测变化；若日报没有 D3 预测，则降级读取 `brent_change_3d`，再没有则读取 `brent_change_1d`。

| 因子组 | 封顶 | 使用字段 | 打分逻辑 |
| --- | ---: | --- | --- |
| 成本侧 | ±35 | `brent_change_usd_d3` | ≥3 美元 +35；2~3 +25；1~2 +15；0.5~1 +5；-0.5~0.5 为 0；-1~-0.5 -5；-2~-1 -15；-3~-2 -25；≤-3 -35 |
| 供应侧 | ±35 | `shandong_cdu_utilization_percentile_weekly` | 开工率分位 <25 +35；25~45 +18；45~55 为 0；55~75 -18；>75 -35 |
| 供应侧修正 | ±5 | `shandong_refinery_load_news_adjustment_d3` | 临时降负、检修、装置扰动等供应新闻修正；缺失按 0 |
| 需求侧 | ±20 | `sales_production_ratio_d3_avg` | 三日产销率均值 ≥110% +20；100~110 +10；95~100 +10；90~95 +5；85~90 为 0；70~85 -10；<70 -20 |
| 情绪侧 | ±10 | `trader_sentiment_label_d3` | 成交活跃/抢货 +10；中性 0；抛货/让利 -10 |

### 3. W1 业务基准规则

W1 判断未来一周的价格偏向，Brent 使用 `horizon_forecasts.W1`；若缺失，则降级读取 `brent_change_5d`、`brent_change_3d`。

| 因子组 | 封顶 | 使用字段 | 打分逻辑 |
| --- | ---: | --- | --- |
| 成本侧 | ±30 | `brent_change_usd_w1` | ≥5 美元 +30；3~5 +20；1.5~3 +10；0.5~1.5 +5；-0.5~0.5 为 0；-1.5~-0.5 -5；-3~-1.5 -10；-5~-3 -20；≤-5 -30 |
| 供应侧 | ±30 | `shandong_cdu_utilization_percentile_weekly` | 开工率分位 <25 +30；25~45 +15；45~55 为 0；55~75 -15；>75 -30 |
| 供应侧修正 | ±5 | `shandong_refinery_load_news_adjustment_w1` | 一周内临时降负、检修、装置扰动修正；缺失按 0 |
| 需求侧 | ±20 | `sales_production_ratio_w1_avg` | 周度产销率均值 ≥110% +20；100~110 +10；95~100 +10；90~95 +5；85~90 为 0；70~85 -10；<70 -20 |

| 政策预期 | ±5 | `price_window_expectation_weekly` | 上调预期 +5；中性 0；下调预期 -5 |

W1 库存口径按业务要求优先使用：

```text
贸易商库存 + 主营库存 + 独立炼厂库存合计
```

若组件不完整，则该项按缺失处理，不用单一库存字段硬代替。

| --- | --- |

| 山东现货跳变 | 只修正 D1 点位，不直接修正 D3/W1/M1 点位 |

### 6. 业务基准点位

业务基准分本身不是价格，也不再用旧线性换算。它使用和智能体综合预测相同的“状态桶到历史分布”框架，但不共用同一套阈值。原因是两者不是同一种分数体系：

| 项目 | 智能体综合预测 | 业务基准模型 |
| --- | --- | --- |
| 分数来源 | 多个规则智能体按有效满分归一化后加权 | 《山东成品油市场价预测打分模型》逐项相加 |
| 分数形态 | -1 到 +1 的综合小数，进入桶前乘 100 | -100 到 +100 的业务原始总分 |
| 状态桶阈值 | 使用综合分强弱阈值 | 使用业务总分强弱阈值 |
| 样本不足处理 | 可逐步合并更多相邻桶 | 只允许同桶或相邻桶；仍不足则走专家冷启动，避免用弱信号样本稀释强业务分 |

业务基准模型状态桶如下：

| 业务状态桶 | 业务总分范围 | 业务含义 |
| --- | ---: | --- |
| 强空 | <= -70 | 业务规则显著偏空 |
| 偏空 | -70 到 -35 | 业务规则偏空 |
| 弱空 | -35 到 -10 | 业务规则小幅偏空 |
| 震荡 | -10 到 10 | 业务规则方向不明显 |
| 弱多 | 10 到 35 | 业务规则小幅偏多 |
| 偏多 | 35 到 70 | 业务规则偏多 |
| 强多 | >= 70 | 业务规则显著偏多 |

换算路径为：

```text
业务总分 → 状态桶 → 同桶历史真实涨跌 P50 中位数 → 业务基准预测涨跌额
业务基准预测点位 = 当前山东 92# 市场价 + 业务基准预测涨跌额
```

例如业务总分为 70 分时，命中的是业务基准模型的“强多”桶 `>=70`，不是智能体综合预测的“强多”桶 `>=12`。如果业务强多桶当前历史样本不足，系统不会拿 15 分、20 分这类弱多样本硬凑；会走专家冷启动状态桶，当前 D1 冷启动强多基准涨跌为 `+45 元/吨`。

业务基准模型只作为独立对照，不参与智能体综合分和综合可靠度。

## 六、方向判断

方向不是直接看分数正负，而是看预测涨跌额是否超过当前周期的最小展示幅度。

| 周期 | 当前最小展示幅度 | 含义 |
| --- | ---: | --- |
| D1 | 3 元/吨 | 小于该幅度时按震荡展示 |
| D3 | 8 元/吨 | 小于该幅度时按震荡展示 |
| W1 | 12 元/吨 | 小于该幅度时按震荡展示 |
| M1 | 32 元/吨 | 小于该幅度时按震荡展示 |

这组数只是方向展示的“死区”，不是点位生成参数。它的作用是避免几元钱的微小变化被展示成明确上涨或明确下跌。

## 七、区间生成

系统区间围绕预测点位生成：

```text
预测区间 = 预测点位 ± 区间半宽
```

区间半宽由以下因素共同决定：

| 因素 | 影响 |
| --- | --- |
| 同桶历史分布 | 同状态历史涨跌越分散，区间越宽 |
| 周期长短 | 周期越长，基础区间越宽 |
| 数据质量 | 数据缺失或使用兜底数据时，区间扩大 |
| 因子分歧 | 多个智能体方向冲突时，区间扩大 |
| 事件风险 | 事件风险越高，区间扩大 |

当前周期基础半宽：

| 周期 | 基础半宽 |
| --- | ---: |
| D1 | 50 元/吨 |
| D3 | 85 元/吨 |
| W1 | 150 元/吨 |
| M1 | 280 元/吨 |

基础半宽不是点位换算参数，只是风险展示的下限。若同桶历史分布或事件风险更宽，系统会扩大区间。

## 八、研判可靠度

研判可靠度不是价格命中概率，而是“这次结论能不能作为经营参考”的质量评分。

它由五部分构成：

| 组成 | 含义 |
| --- | --- |
| 信号强度 | 预测涨跌额相对区间半宽是否足够明显 |
| 因子一致性 | 多个规则智能体是否支持同一个方向 |
| 分桶样本质量 | 当前状态桶或合并桶样本是否足够 |
| 数据质量 | 数据源是否完整，各智能体是否有缺失字段 |
| 事件稳定性 | 是否存在黑天鹅、地缘、政策突发等扰动 |

业务基准模型不参与研判可靠度计算，只作为独立对照。

## 九、经营建议生成

经营建议不是大模型自由发挥，而是先由规则根据以下因素生成：

```text
方向判断、预测涨跌额、区间半宽、研判可靠度、事件风险、调价窗口、核心驱动
```

建议分为几类：

| 场景 | 建议方向 |
| --- | --- |
| 明确上涨且可靠度较高 | 逢低补库、控制低价放量、缩短报价有效期 |
| 明确下跌且可靠度较高 | 以销定采、控制高价库存、弱势区域加快成交 |
| 震荡或可靠度一般 | 维持滚动采购和安全库存，不做方向性重仓 |
| 事件扰动较高 | 降低动作强度，等待人工确认 |
| 极高事件风险 | 暂停自动经营动作，切人工复核 |

每条建议必须说明：

```text
建议动作、触发原因、执行幅度、止损条件、责任角色
```

大模型只负责把规则结果表达得更清楚，或补充复核问题；不允许直接修改价格点位、区间和方向。

## 十、裂解价差

汽油裂解价差按税后现货口径计算：

```text
汽油裂解价差 = 山东92#市场价 / 1.13 - 汽油消费税 - Brent价格 × 吨桶比 × 人民币汇率中间价
```

当前系统口径：

| 项目 | 当前取值 |
| --- | --- |
| 汽油消费税 | 2109.76 元/吨 |
| 吨桶比 | 6.77 |
| 人民币汇率 | 人民币汇率中间价 |
| 当前裂解 | 使用当前山东 92# 价格和当前 Brent |
| 预测裂解 | 使用预测山东 92# 点位和 Brent 预测点位 |

因此，有了山东 92# 预测点位和 Brent 预测点位后，系统可以同步生成汽油裂解价差预测值。

## 十一、区域价差和净回款

区域价差按“目标区域价格 - 山东价格”计算：

```text
区域价差 = 目标区域 92# 价格 - 山东 92# 价格
净回款价差 = 区域价差 - 运费
```

当前经营判断只使用净回款价差，不计算损耗、装卸、资金成本、信用缓冲等暂时没有结构化来源的数据。

### 11.1 区域价差预测逻辑

区域价差预测是独立预测模块，不直接套用山东 92# 绝对价格的涨跌结论。系统先预测“目标区域相对山东的价差会走扩还是收敛”，再和山东 92# 预测价组合，得到目标区域的预测价格。

```text
预测区域价差 = 当前区域价差 + 预测价差变化量
目标区域预测价 = 山东 92# 预测价 + 预测区域价差
预测净回款价差 = 预测区域价差 - 运费
```

当前支持的区域包括华东、华北、华南、华中、西北、西南、东北。区域价差口径统一为：

```text
目标区域 92# 汽油价格 - 山东 92# 汽油价格
```

也就是说，区域价差为正，表示目标区域价格高于山东；区域价差为负，表示目标区域价格低于山东。

#### 输入数据

| 数据 | 用途 |
| --- | --- |
| 山东 92# 汽油市场价 | 作为山东基准价 |
| 目标区域 92# 汽油市场价 | 与山东价格计算当前区域价差 |
| 区域价差历史序列 | 只用于计算当前 1 日、3 日价差变化，判断短线价差动量；不再用历史回归系数拟合预测涨跌 |
| 区域运费 | 计算净回款价差；人工录入后持续沿用 |
| 区域库存数据 | 判断目标区域库存压力，来自隆众已购库存数据 |
| 成品油资讯和事件标签 | 有明确区域影响时作为辅助信号；无明确区域影响时不强行打分 |

#### 三个区域价差智能体

区域价差模块当前由三个专门智能体给出信号：

| 智能体 | 判断问题 | 主要规则 |
| --- | --- | --- |
| 价差结构智能体 | 当前价差本身是在走扩还是收敛 | 看区域价差 1 日和 3 日变化；价差连续走扩则偏走扩，连续收窄则偏收敛；价差过高时提示回落压力，价差过低时提示修复空间 |
| 运费净回款智能体 | 扣除运费后是否仍有跨区外发经济性 | 净回款价差越高，说明外发空间越好，但过宽时也会提示后续收敛压力；净回款为负时，原则上不支持外发 |
| 区域库存压力智能体 | 目标区域是否因为库存高低导致价格强弱变化 | 目标区域库存高于同类区域中位水平时，目标区域价格承压，价差倾向收敛；库存低于中位水平时，目标区域抗跌，价差倾向走扩 |

#### 点位生成

区域价差点位采用“冻结状态表中位数 + 当日规则修正 + 经营边界约束”的组合方法。三个区域价差智能体仍然负责给出证据，但预测涨跌不使用历史回归系数、截距或斜率拟合。

```text
预测价差变化量 =
状态表价差变化中位数
+ 当日规则修正 × 修正权重

预测区域价差 =
clip(
  当前区域价差 + 预测价差变化量,
  经营下边界,
  经营上边界
)
```

上面公式里的 `clip` 是限幅函数，含义是“把预测结果限制在经营允许的上下边界内”。公式可以理解为：

```text
如果计算出的预测区域价差 < 经营下边界，则按经营下边界展示；
如果计算出的预测区域价差 > 经营上边界，则按经营上边界展示；
如果计算出的预测区域价差在上下边界之间，则保留原计算结果。
```

举例：

```text
当前区域价差 + 预测价差变化量 = 180
经营下边界 = 60
经营上边界 = 150

clip(180, 60, 150) = 150
```

也就是说，模型原始算出 180 元/吨，但超过了当前线路和历史分布能解释的上边界，最终只展示 150 元/吨，避免给出经营上明显不合理的点位。

这里预测的是区域价差自身变化，不是山东 92# 绝对价格。因此，区域价差预测不会因为山东价格预测上涨就机械地认为所有区域也同步上涨，而是单独判断各区域相对山东的强弱。

状态表只使用预测日前已经能验证完成的样本。例如预测 D1 时，历史样本必须满足“历史日期 + 1 个交易日 <= 当前预测日”，避免把未来数据混进来。

当前状态表分组如下：

| 状态维度 | 分组方式 |
| --- | --- |
| 价差位置 | 本区域历史价差 P10/P30/P70/P90，分为极低、低位、中性、高位、极高 |
| 价差动量 | 3 日价差变化 >=2 为走扩，<=-2 为收敛，其余为稳定 |
| 净回款状态 | 净回款 >=80 为强顺挂，30-80 为顺挂，0-30 为弱顺挂，<0 为关闭 |

P10、P30、P70、P90 是历史分位数，不是价格点位规则。计算方法是：把本区域历史价差从低到高排序，P10 表示 10% 的历史样本低于这个价差，P90 表示 90% 的历史样本低于这个价差。

举例：某区域过去 100 天价差从低到高排序后：

```text
第10个样本对应的价差 = P10
第30个样本对应的价差 = P30
第70个样本对应的价差 = P70
第90个样本对应的价差 = P90
```

系统用这些分位数判断当前价差位置：

| 当前价差位置 | 状态含义 |
| --- | --- |
| <= P10 | 极低价差，理论上有修复空间 |
| P10-P30 | 低位价差，偏修复 |
| P30-P70 | 正常价差区间 |
| P70-P90 | 高位价差，偏收敛 |
| >= P90 | 极高价差，收敛风险更高 |

P5、P95 也类似，只是用于经营上下边界，代表更极端的历史低位和高位。

系统优先查“价差位置 + 动量 + 净回款状态”的完整状态；如果样本不足，逐级降为“价差位置 + 动量”、再降为“价差位置”。每级状态至少需要 12 条有效样本，否则降级为当日规则修正。

当日规则修正如下：

| 贡献项 | 计算方式 | 经营含义 |
| --- | --- | --- |
| 价差动量贡献 | `3日价差变化 × 0.45 + 1日价差变化 × 0.25`，限幅 ±6 元/吨 | 价差连续走扩则顺势上修，连续收窄则顺势下修 |
| 价差极值回归贡献 | 当前价差 ≥160 扣 5，120-160 扣 3，80-120 扣 1；当前价差 ≤0 加 4，0-30 加 2 | 价差处于极端高位时防回落，处于极端低位时看修复 |
| 净回款套利贡献 | 净回款 ≥80 扣 4，40-80 扣 2，0-40 扣 1；净回款为负不直接修正点位 | 运费后套利窗口过宽时，后续容易被流向修复压缩 |
| 区域库存贡献 | 库存/同类区域中位数 ≥1.25 扣 4，1.12-1.25 扣 2；≤0.75 加 4，0.75-0.88 加 2；库存环比 ±5 万吨以上再加减 1.5 | 目标区域高库存压制价格，低库存支撑区域抗跌 |

其中 `3日价差变化 × 0.45 + 1日价差变化 × 0.25` 是当前试运行规则中的短期动量修正，不是最终业务确认参数。设计原则如下：

| 参数 | 设计原因 |
| --- | --- |
| 3 日价差变化权重 0.45 | 3 日变化比单日变化更能代表短线趋势，权重高于 1 日变化 |
| 1 日价差变化权重 0.25 | 1 日变化用于确认最新变化，但容易受单日噪声影响，所以权重较低 |
| 0.45 + 0.25 < 1 | 不让动量项完全等同于历史涨跌本身，避免短期波动被模型放大 |
| 限幅 ±6 元/吨 | 动量只是当日修正项，不能单独决定区域价差点位，防止状态表中枢被短期噪声覆盖 |

举例：

```text
3日价差变化 = -3.77
1日价差变化 = -1.30

价差动量贡献 = -3.77 × 0.45 + -1.30 × 0.25
             = -1.6965 + -0.325
             = -2.0215 元/吨
```

如果计算结果超过 ±6，则按 ±6 处理：

```text
计算结果 = +9
限幅后 = +6

计算结果 = -8
限幅后 = -6
```

0.45、0.25、±6 的当前设定不是说 0.55、0.35 一定不可以，而是选择了一组偏保守的试运行参数。选择依据是：

1. 当前系统可用区域价差样本中，1 日价差变化通常很小，所有区域合并后 1 日绝对变化中位数约 0.88 元/吨，P90 约 1.33 元/吨。
2. 3 日价差变化更能代表短线趋势，所有区域合并后 3 日绝对变化中位数约 2.64 元/吨，P90 约 3.97 元/吨。
3. 采用 0.45、0.25 后，动量修正的 P90 约 2.12 元/吨，属于温和修正；如果改成 0.55、0.35，P90 会提高到约 2.65 元/吨，模型会更敏感、更容易追随短期波动。
4. 状态表启用后，规则修正还要乘 35%，因此 0.45、0.25 对最终点位的常规影响大约在 1 元/吨以内，主要起到“微调中枢”的作用。
5. ±6 元/吨限幅约为当前动量修正常规 P90 的 2.8 倍，只在极端价差变化时触发，目的是防止单日或短期异常波动把点位拉偏。

因此，当前参数的业务含义是：**宁可让动量项少修一点，也不要让短期波动覆盖状态表中枢。** 如果业务希望模型对区域价差短线变化更敏感，可以把 0.45、0.25 调高到 0.55、0.35，但必须重新冻结规则并做前推验证，确认更高权重确实降低误差，而不是放大噪声。

状态表启用时，当日规则修正只按 35% 权重进入点位，避免短期噪声覆盖状态表中枢；状态表样本不足时，规则修正按 100% 权重作为降级方案。

35% 权重的含义是：状态表中位变化是主判断，当日规则修正只作为辅助修正。当前没有把 35% 作为统计最优参数，而是按“主模型约占 2/3、短期修正约占 1/3”的试运行原则设定。后续如果前推验证发现短期规则修正长期有效，可以提高权重；如果短期修正噪声较大，则降低权重。

多周期规则修正不是重新拟合，而是在 D1 贡献基础上乘周期倍数并限幅：

| 周期 | 周期倍数 | 变化量限幅 |
| --- | ---: | ---: |
| D1 | 1.0 | ±12 元/吨 |
| D3 | 1.6 | ±22 元/吨 |
| W1 | 2.4 | ±35 元/吨 |
| M1 | 4.0 | ±50 元/吨 |

经营边界约束由本区域历史价差分布和线路成本共同确定：

```text
分布下边界 = 历史P5 - 0.5 × 稳健波动
分布上边界 = 历史P95 + 0.5 × 稳健波动
线路下边界 = -0.35 × 运费 - 风险缓冲
线路上边界 = 运费 + 必要毛利 + 风险缓冲

经营下边界 = max(分布下边界, 线路下边界)
经营上边界 = min(分布上边界, 线路上边界)
```

示例：某区域当前价差 73.66，状态表同状态 D1 价差变化中位数为 -0.94；当日规则修正为 -6.02，修正权重为 35%，则：

```text
预测变化量 = -0.94 + 0.35 × -6.02 = -3.04
预测区域价差 = 73.66 - 3.04 = 70.62
```

需要特别说明：区域价差不使用历史回归系数拟合点位。后续验证仍必须使用冻结规则后的前推验证：固定某一天以前的规则，只用之后真实发生的数据检验方向、点位误差和净回款判断。

#### 区间和方向

区域价差区间使用周期基础半宽、因子分歧和数据缺口共同生成，并设置经营展示上限，避免区间无限放大。当前区间是风险提示口径，不是已经被独立验证过的命中区间。方向判断按“预测价差变化量”判断：

| 方向 | 含义 |
| --- | --- |
| 走扩 | 预测区域价差上升，目标区域相对山东更强 |
| 收敛 | 预测区域价差下降，目标区域相对山东转弱 |
| 震荡 | 预测变化量较小，暂不形成明确价差信号 |

当前区域价差结果只能作为试运行观察，不能直接宣称为经营可用准确率。后续应按“冻结参数 + 前推验证 + 与简单基准模型对比”的方式重新评估。

| 净回款价差 | 经营含义 |
| ---: | --- |
| 大于 60 元/吨 | 外发经济性较强，可考虑扩大外发 |
| 30-60 元/吨 | 小批量试发或锁定订单 |
| 0-30 元/吨 | 以客户维护和结构性订单为主 |
| 小于 0 元/吨 | 原则上不建议外发 |

运费由人工录入，保存后持续沿用；不修改时，系统使用上一笔有效运费。
