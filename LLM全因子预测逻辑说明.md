# LLM全因子预测逻辑说明

## 1. 目的
用 LLM 在不替代现有规则模型的前提下，基于同一批可见因子，给出新的山东92#汽油预测结果、点位、区间和研判逻辑。

## 2. 输入因子

### 2.1 成品油侧
- 山东92#现货价
- 全国92#现货价
- 各区域92#价格
- 区域价差（目标区域价格 - 山东价格）
- 山东汽油产销率
- 山东地炼开工率
- 山东汽油库存
- 山东汽油库容率
- 山东炼油利润
- MTBE、石脑油价格
- 汽油裂解价差

### 2.2 原油侧
- Brent 结算价
- Brent 日度预测
- Brent W1-W4 周度预测
- Brent 实时盘面回吐/反弹信息
- Brent 报告中的多空命中数、方向和核心驱动

## 3. 提示词设计原则

### 3.1 基本要求
- 只使用 source_date 及之前的数据
- 不允许读取 target_date 实际价格
- 输出必须是 JSON
- 方向、点位、区间、置信度必须同时给出

### 3.2 研判流程
1. 先识别成本、需求交投、库存供应、区域价差、政策天花板五类驱动
2. 再把驱动强弱转换成预计涨跌额
3. 再由当前价 + 预计涨跌额得到点位
4. 最后根据历史波动和驱动一致性给出区间

## 4. 量化映射

### 4.1 综合分到涨跌额
- `net_score <= -7`：`-100 ~ -160`
- `-6 ~ -4`：`-50 ~ -100`
- `-3 ~ +3`：`-30 ~ +30`
- `+4 ~ +6`：`+40 ~ +90`
- `net_score >= +7`：`+90 ~ +160`

### 4.2 强上涨触发
满足以下条件时，不应给出过小涨幅：
- Brent 反弹超过 `3 USD/bbl`
- 产销率 `>= 90`
- 汽油库容率 `< 40%`

若同时满足，通常不应低于 `+80`，除非另有两项以上重大利空。

## 5. 原油输入如何使用

原油不只看一个价格点，而是组合输入：
- 当前 Brent 结算价
- 当日 Brent 预测
- W1-W4 周度预测
- 实时盘面回吐/反弹
- 核心驱动说明

用途：
- 判断成本端方向
- 判断成本端是“已经落定”还是“还有后续推力”
- 防止把一次跳涨过度放大成次日继续猛涨

## 6. 输出字段

- `direction`
- `driver_scores`
- `net_score`
- `expected_change_yuan_per_ton`
- `point_forecast`
- `range_low`
- `range_high`
- `confidence`
- `reasoning_logic_zh`
- `risk_note_zh`

## 7. 6月3日回测示例

### 输入日
- `2026-06-02`

### 预测目标
- `2026-06-03`

### 结果
- 当前价：`8086`
- 方向：`下跌`
- 预计涨跌额：`-30`
- 点位：`8056`
- 区间：`8020 ~ 8086`
- 置信度：`80`

### 逻辑摘要
- 原油 W1 仍偏多，但当日原油盘面已出现回吐，成本端只给轻微支撑
- 6月2日现货大涨后，6月3日下游抵触情绪上升，产销率从 `96%` 降至 `62%`
- 山东汽油库存偏低，限制下跌空间
- 区域套利窗口关闭，山东价格在大涨后失去外流优势

## 8. 结论
这套 LLM 逻辑适合做“第二套研究结果”，重点看方向、逻辑和对行情变化的解释。
要想稳定到可用，关键不是继续堆因子，而是把：
- 输入边界固定
- 提示词固定
- 输出 JSON 固定
- 历史回测固定

这样才能和现有规则模型做可比对、可回放的结果。
