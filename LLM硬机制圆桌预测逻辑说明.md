# LLM硬机制圆桌预测逻辑说明

## 1. 定位
这套逻辑不是规则模型，也不是让 LLM 按五类因子打分。

它的目标是模拟真实成品油研究圆桌：
- 专家先判断市场行为
- 交叉质询再找证据弱点
- 主持人最后用价格调整台账推导点位

## 2. 三阶段流程

### 2.1 第一阶段：专家独立判断
6 位专家分别从不同交易视角判断目标日行情：

- 山东地炼报价负责人
- 下游贸易商与终端采购
- 区域套利物流专家
- 原油与调和成本专家
- 供应库存专家
- 反方审稿专家

第一阶段硬约束：
- 禁止输出点位
- 禁止输出区间
- 禁止输出涨跌金额
- 禁止打分
- 只能输出交易情景、方向倾向、核心证据、反证和盲点

程序会校验字段。
如果专家阶段出现 `point`、`range`、`change`、`score`、`forecast`、`expected` 等字段，直接判定不合规并重跑。

### 2.2 第二阶段：交叉质询
交叉质询专家只做四件事：
- 找最强共识
- 找核心分歧
- 找证据弱点
- 找角色偏见

第二阶段仍然禁止输出价格点位和涨跌金额。

### 2.3 第三阶段：主持人定价
主持人最后才允许输出价格。

主持人不能机械平均专家意见，必须先判断主导情景：
- 走量修复
- 成本托底
- 区域外流
- 高位僵持
- 继续推涨

然后用价格调整台账从当前价推导点位。

## 3. 价格调整台账

台账不是因子分数，而是交易行为修正：

- 成交修复折让
- 区域外流修复折让或溢价
- 成本托底修正
- 库存/利润托底修正
- 风险缓冲修正

最终要求：

```text
最终点位 = 当前价 + 台账各项调整合计
```

程序会校验：
- 台账合计必须等于 `total_adjustment`
- 当前价加合计必须等于 `final_point`
- 区间必须包含最终点位

## 4. 2026-06-03 离线试验结果

输入日：`2026-06-02`

预测日：`2026-06-03`

当前山东92#：`8086 元/吨`

最终结果：

```text
方向：下跌
预计调整：-45 元/吨
点位：8041 元/吨
区间：8020 - 8060 元/吨
```

价格调整台账：

```text
成交修复折让：-35
区域外流修复折让：-25
成本托底修正：-10
库存/利润托底修正：+25
风险缓冲修正：0
合计：-45
```

## 5. 与规则模型的区别

规则模型回答：

```text
因子分数支持什么方向？
```

硬机制圆桌回答：

```text
明天真实市场参与者会怎么报价、接货、外流和博弈？
```

这套机制更像业务讨论，不是因子加权。

## 6. 输出文件

离线脚本会生成两个文件：

- 原始 JSON：保存完整 prompt 输出、专家意见、质询结果、主持人结果
- Markdown 摘要：用于业务汇报和快速阅读

示例：

```powershell
python scripts\llm_roundtable_forecast.py --source-date 2026-06-02 --target-date 2026-06-03
```

输出：

```text
artifacts/llm_roundtable_20260602_to_20260603.json
artifacts/llm_roundtable_20260602_to_20260603.md
```
