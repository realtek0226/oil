# 价格研判与经营建议逻辑说明

本文用于向业务人员说明系统如何形成价格研判、经营建议和区域价差判断。

## 一、系统定位

当前系统不是“大模型直接预测油价”，而是规则智能体、大模型标签抽取和历史样本映射共同工作的研究台。

系统输出三类结果：

| 能力 | 说明 |
| --- | --- |
| 价格研判 | 给出山东 92# 汽油 D1、D3、W1、M1 的参考点位、区间和方向 |
| 经营建议 | 判断本次结论能不能用、适合什么动作、什么情况下必须降级或人工复核 |
| 区域观察 | 展示山东与各区域价差、运费和净回款价差 |

大模型不直接决定价格点位。价格点位和方向由规则打分、历史同状态样本分布和强信号修正生成；大模型主要用于成品油资讯标签抽取、事件归因、原因表述和复核提示。

## 二、价格研判主链路

```text
山东现货价和各类因子数据
        ↓
各规则智能体分别打分
        ↓
形成智能体综合分和业务基准分
        ↓
分数进入市场状态桶
        ↓
查找历史同状态桶的真实涨跌分布
        ↓
取历史中位数作为基础涨跌额；方向桶被抵消时，启用同方向保守分位数
        ↓
叠加山东现货跳变、高等级事件等明确强信号修正
        ↓
生成预测点位、预测区间、方向、可靠度和经营建议
```

系统已经取消两种旧做法：

| 旧做法 | 为什么不用 |
| --- | --- |
| 旧线性换算 | 样本阶段不足时容易出现负分映射成上涨，业务解释困难 |
| 综合分 × 人工点值 | 点值来源无法业务追溯，设成 10 或 80 都缺少依据 |

当前采用的是“分数分桶后的历史同状态分布映射”。分数只表示市场状态强弱，价格涨跌额来自历史上同类状态实际发生过的涨跌分布。

## 三、当前使用的数据

| 数据类别 | 用途 |
| --- | --- |
| 山东 92# 汽油市场价 | 当前价格锚点，预测点位以它为基础 |
| Brent 结算价和 Brent 预测日报 | 判断原油成本变化和未来成本压力 |
| 汽油裂解价差、MTBE、石脑油、汇率 | 判断成本和调油支撑 |
| 山东地炼开工率、炼油利润、库存 | 判断供应压力 |
| 山东地炼产销率 | 判断需求承接强弱 |
| 全国及区域 92# 汽油价格 | 判断区域价差和跨区外发机会 |
| 发改委调价窗口、调价预期金额 | 判断政策周期扰动 |
| 成品油资讯、山东日评、事件快讯 | 判断成交活跃度、贸易商心态、突发事件风险 |
| 区域运费 | 计算净回款价差 |

早报场景的数据边界为：

```text
预测日 T 的早报，只使用 T 日 07:00 前已经可见或已经落库的数据。
```

T 日 07:00 后出现的资讯进入盘中预警，不倒灌到早上的预测中。

Brent 预测日报是 D1 点位的重要输入。若预测日没有同日 Brent 日报，系统标记为 `brent_daily_report_missing`；若接口返回旧日期日报，系统标记为 `brent_daily_report_stale`。这两类场景都会作为降级参考：点位仍可展示，但可靠度压低，区间明显放宽，经营建议不能按正常可执行结论处理。

## 四、智能体分别判断什么

| 智能体 | 业务含义 | 主要看什么 |
| --- | --- | --- |
| 原油成本 | 原油和调油成本是否支撑涨跌 | Brent 预测涨跌、MTBE、石脑油；裂解价差仅观察 |
| 市场结构 | 山东价格相对全国和区域是否偏强或偏弱 | 山东与全国、华东、华北、华南、华中、西北、西南、东北价差 |
| 供给库存 | 供应端是否偏紧或偏宽松 | 开工率、库存、炼油利润 |
| 需求季节性 | 需求承接是否足够强 | 产销率、月度产销变化、淡旺季、节假日 |
| 政策周期 | 发改委调价窗口是否放大现货波动 | 调价预期金额、窗口天数、上轮调价 |
| 成品油资讯 | 现货交易氛围是否变化 | 成交活跃度、贸易商心态、报价行为 |
| 山东现货跳变 | 是否有强烈跳涨或跳跌信号 | 实际成交重心、炼厂推价、低价资源、封单惜售、抢货/抛货 |
| 事件风险 | 是否存在突发风险导致结论降级 | 地缘、OPEC、航运、制裁、黑天鹅 |

每个智能体只输出自己的分数、方向和依据。不同智能体的原始有效满分不完全一样，例如原油成本当前是 -65 到 +65，供给库存当前是 -62 到 +62；系统会先按各自有效满分归一化，再按专家权重合成综合分。最终点位由统一的分桶映射层生成，不由单个智能体直接决定。

## 五、点位如何生成

系统先把各智能体原始分按有效满分归一化，再按专家权重合成统一综合分。综合分本身是 -1 到 +1 的小数；进入市场状态桶时换算成“分数点”：

```text
分数点 = 综合分 × 100
```

这里的 ×100 只是单位换算，便于和状态桶阈值比较，不是再次加权，也不是把预测力度放大。

下面这套状态桶只用于智能体综合预测。这里的分数是 `综合分 × 100` 后的分数点，不是业务基准模型的原始总分。

| 智能体综合状态桶 | 分数范围 | 业务含义 |
| --- | ---: | --- |
| 强空 | <= -12 | 多个因素明显压制现货 |
| 偏空 | -12 到 -6 | 下行因素占优 |
| 弱空 | -6 到 -2 | 偏弱但力度有限 |
| 震荡 | -2 到 2 | 方向不明显 |
| 弱多 | 2 到 6 | 偏强但力度有限 |
| 偏多 | 6 到 12 | 上行因素占优 |
| 强多 | >= 12 | 多个因素明显支撑现货 |

点位生成规则：

```text
基础预测涨跌额 = 当前状态桶历史真实涨跌的中位数
如果方向桶中位数被正负样本抵消为 0，且同方向样本充足：
    多头桶取同桶上涨样本 P25
    空头桶取同桶下跌样本 P75
预测点位 = 当前山东 92# 市场价 + 基础预测涨跌额 + 明确强信号修正
```

这里的“历史真实涨跌中位数”就是 P50。它的计算口径是：

```text
历史日真实涨跌 = 目标日山东92#市场价 - 历史日山东92#市场价
P50 = 当前状态桶内这些真实涨跌从低到高排序后的中间值
```

例如，当前命中“偏空”桶，系统会找历史上同样命中偏空桶的样本，计算这些样本之后对应周期的真实涨跌。如果这些真实涨跌排序后中间值是 `-15 元/吨`，那么基础预测涨跌额就是 `-15 元/吨`。如果偏空桶样本不足 12 条，智能体综合预测会合并相邻桶后再取 P50。

如果当前命中“强多、偏多、弱多”这类多头桶，但同桶历史样本里上涨和下跌互相抵消，导致整体 P50 变成 0，系统会再看同桶上涨样本是否足够。样本足够时，取上涨样本的 P25 作为保守上涨幅度；空头桶同理，取下跌样本的 P75 作为保守下跌幅度。这样可以避免多周期预测全部退回当前价。

需要说明：历史桶样本是用当前规则对历史数据重新回放得到的，不等同于历史上真实发布过的预测记录。因此它是“当前规则下的历史同状态回放 P50”，后续应继续用冻结规则做前推验证。

历史样本使用规则：

| 情况 | 系统处理 |
| --- | --- |
| 当前状态桶样本不少于 12 条 | 只用同桶历史样本 |
| 当前状态桶样本少于 12 条 | 合并相邻状态桶，直到样本达到最低要求 |
| 全部相邻桶合并后仍不足 12 条 | 使用可用合并样本，并在页面展示样本量；完全没有可用历史时才使用冷启动专家状态桶 |

方向约束：

| 当前分数状态 | 点位约束 |
| --- | --- |
| 负分状态桶 | 基础涨跌额不能映射成上涨 |
| 正分状态桶 | 基础涨跌额不能映射成下跌 |
| 震荡状态桶 | 基础涨跌额按 0 处理 |
| 方向桶被抵消为 0 | 同方向样本充足时启用保守分位数，样本不足时仍按 0 处理 |

山东现货跳变、高等级事件冲击等强信号可以单独修正 D1 点位，但不允许无限放大。山东现货跳变必须有当日急涨成交验证不足、急涨后当日实质回落，或实际成交/炼厂报价/低价资源/贸易商行为等硬证据；高等级事件只在 Brent 明显波动且事件方向与 Brent 方向一致时做有限对冲，避免和成本智能体重复计分。

## 六、业务基准模型如何使用

业务基准模型来自《山东成品油市场价预测打分模型》转成的结构化规则。它和智能体综合预测是两套独立结果。

| 项目 | 智能体综合预测 | 业务基准模型 |
| --- | --- | --- |
| 定位 | 系统主预测 | 业务原始规则口径对照 |
| 是否相互加权 | 否 | 否 |
| 是否单独出点位 | 是 | 是 |
| 是否参与综合可靠度 | 是 | 否 |

业务基准模型也使用“状态桶到历史分布”的换算框架，但它不和智能体综合预测共用同一套阈值。原因是两者不是同一种分数体系：智能体综合分是归一化加权后的综合分，业务基准分是业务表格逐项相加后的原始总分。

| 业务状态桶 | 业务总分范围 | 含义 |
| --- | ---: | --- |
| 强空 | <= -70 | 业务规则显著偏空 |
| 偏空 | -70 到 -35 | 业务规则偏空 |
| 弱空 | -35 到 -10 | 业务规则小幅偏空 |
| 震荡 | -10 到 10 | 业务规则方向不明显 |
| 弱多 | 10 到 35 | 业务规则小幅偏多 |
| 偏多 | 35 到 70 | 业务规则偏多 |
| 强多 | >= 70 | 业务规则显著偏多 |

业务基准换算路径为：

```text
业务总分 → 市场状态桶 → 同桶历史真实涨跌中位数 → 业务基准预测点位
```

例如业务总分为 70 分时，命中业务基准模型的“强多”桶 `>=70`。它不会再套用智能体综合预测的 `>=12` 强多桶，也不会把 15 分、20 分这类弱业务多头样本混进去硬算。如果业务强多桶及相邻桶样本不足，系统会走专家冷启动状态桶，并在页面显示样本量。

因此，业务基准负分不会再因为旧线性换算被映射成上涨。如果业务基准和智能体综合预测方向不一致，系统会展示差异，供业务人员复核。

## 七、区间和可靠度怎么理解

预测区间不是承诺“价格一定落在这里”，而是经营参考范围。

区间来源分两层：

| 层级 | 说明 |
| --- | --- |
| 分桶区间 | 当前状态桶历史真实涨跌的 P25/P75 或合并桶分位数 |
| 风险扩展区间 | 在分桶区间基础上叠加周期长度、数据缺失、因子分歧和事件风险 |

区间会根据以下因素变宽：

| 因素 | 影响 |
| --- | --- |
| 周期变长 | D3、W1、M1 的区间自然比 D1 更宽 |
| 同桶历史波动大 | 历史同状态涨跌分布越分散，区间越宽 |
| 数据缺失 | 缺失字段越多，区间和可靠度越保守 |
| 因子分歧 | 多个智能体方向冲突时，区间扩大 |
| 事件风险 | 黑天鹅、地缘、政策突发会扩大风险范围 |

研判可靠度不是“价格命中概率”。它表示本次结论作为经营参考的质量，主要看：

```text
信号强度、因子一致性、分桶样本质量、数据质量、事件稳定性
```

可靠度高也不等于一定要重仓操作，还要看预测变化是否足够大、事件风险是否可控、经营可用性等级是否达标。

## 八、经营建议如何生成

经营建议先由规则生成，再由大模型做表达优化和复核提示。

规则先判断：

| 判断项 | 作用 |
| --- | --- |
| 预测方向 | 判断偏涨、偏跌还是震荡 |
| 预测涨跌额 | 判断变化幅度是否值得采取动作 |
| 区间半宽 | 判断点位不确定性有多大 |
| 研判可靠度 | 判断结论质量 |
| 事件风险 | 判断是否需要降级或人工确认 |
| 调价窗口 | 判断是否临近发改委调价扰动 |
| 核心驱动 | 判断动作应该围绕成本、供需、政策、成交还是区域价差 |

经营建议分层：

| 场景 | 建议 |
| --- | --- |
| 明确上涨且可靠度较高 | 逢低补库、控制低价放量、缩短报价有效期 |
| 明确下跌且可靠度较高 | 以销定采、控制高价库存、弱势区域小步快跑成交 |
| 震荡或变化不大 | 维持滚动采购和安全库存，不做方向性重仓 |
| 事件扰动中 | 降低动作强度，等待研究员确认事件传导 |
| 极高事件风险 | 暂停自动经营动作，必须人工复核 |

每条建议必须包含：

```text
建议动作、触发原因、执行幅度、止损条件、责任角色
```

大模型不能改价格、区间和方向，只能把已有结论解释清楚，并补充需要人工复核的问题。

## 九、事件和资讯怎么使用

成品油资讯和事件快讯主要有三类用途：

| 用途 | 说明 |
| --- | --- |
| 交易情绪标签 | 判断成交活跃、观望、抛货让利等状态 |
| 事件风险闸门 | 判断是否存在高风险事件，需要降级或人工确认 |
| 原因解释 | 把“为什么上涨或下跌”说清楚 |

资讯和事件不再用来拟合线性参数。能形成稳定标签并落库的资讯，进入当前智能体打分；没有完整历史标签的部分，不强行反推历史，只在可靠度和风控层体现不确定性。

## 十、区域价差和净回款

区域价差定义为：

```text
区域价差 = 目标区域 92# 汽油价格 - 山东 92# 汽油价格
```

经营上更关注净回款价差：

```text
净回款价差 = 区域价差 - 运费
```

系统当前只展示净回款价差，不计算损耗、装卸、资金成本、信用缓冲等暂时没有结构化来源的数据。

运费由人工录入，保存后持续沿用。如果运费长期未确认，系统会提示复核。

## 十一、当前结论怎么给业务使用

建议业务人员按以下顺序看系统：

1. 先看山东 92# 当前价格、预测点位和预测区间。
2. 再看方向是明确上涨/下跌，还是震荡偏强/偏弱。
3. 看点位映射用的是哪个状态桶、同桶样本有多少条。
4. 看研判可靠度和经营可用性等级，判断能否执行动作。
5. 看核心驱动，确认是原油、供需、政策、资讯还是事件在推动。
6. 看业务基准模型是否与智能体综合预测一致。
7. 最后看区域净回款价差，判断是否有跨区外发机会。

如果系统显示“事件扰动中”“等待人工确认”或数据缺失较多，不建议直接按点位做经营动作，应先由研究员、贸易岗、销售岗或物流岗复核。
